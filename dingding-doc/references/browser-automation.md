# 浏览器自动化与失败恢复

本文件补充 agent 操作钉钉文档 Web 编辑器时的 Playwright、剪贴板、定位、选区、保存状态和恢复策略。先以当前页面真实可见状态为准；任何选择器、文本入口或快捷键失败后都要重新读取页面或截图。

## 基本探测

进入文档后先确认三件事：

1. 外层 URL 是 `alidocs.dingtalk.com`。
2. 页面存在 `iframe#wiki-doc-iframe`。
3. iframe 内 `body.innerText()` 包含正文或工具栏入口。

示例：

```js
const frame = tab.playwright.frameLocator('#wiki-doc-iframe');
const body = frame.locator('body');
const text = await body.innerText({ timeoutMs: 5000 });
```

如果 iframe 不存在、读取超时或正文长期为空：

- 检查外层页面文本是否是登录、无权限、加载失败、企业选择、风险验证或 CAPTCHA。
- 刷新一次后仍失败时，让用户在浏览器中处理阻塞。
- 不要用外层 `document.body.innerText` 代替正文读取。

## 点击与插入点

优先点击 iframe 内的正文区域，再用键盘移动光标。不要只依赖外层坐标。

常用策略：

- 文档末尾：点击正文后按 `ControlOrMeta+End`，再按 `Enter` 新建段落。
- 当前标题后：用 `ControlOrMeta+F` 搜索标题，关闭搜索框后点击命中行末尾，再按 `End` 和 `Enter`。
- 某段前：搜索精确段落片段，点击命中行开头附近，再按 `Home` 和 `Enter`。
- 表格单元格：先截图确认行列，再点击目标单元格中心；粘贴前输入一个短测试字符风险较高，除非用户允许。

定位不确定时，先截图并向用户说明需要其把光标放到目标位置，或改为把内容追加到文档末尾。

## 粘贴载荷

长文本、代码、TSV 和 Mermaid 源码都优先通过剪贴板一次性粘贴，避免逐字符输入导致中文输入法、快捷命令或自动格式干扰。

粘贴前准备：

- 普通长文：Markdown/纯文本。
- 表格：TSV，列之间使用 tab，行之间使用换行。
- 代码：完整代码文本，先创建代码块再粘贴。
- 公式：LaTeX 或用户给出的表达式，先创建公式块或公式入口再粘贴。
- 图形 fallback：Mermaid 源码放入代码块，并说明不是原生渲染图。

粘贴后检查：

- iframe 文本包含关键句、表头、代码函数名或图形源码关键字。
- 截图显示内容没有落入错误位置、错误单元格或普通段落。
- 代码块、表格、公式等结构没有被普通文本化。

## 搜索、选择和替换

修改已有内容前使用最小可验证范围。

可靠顺序：

1. 读取 iframe 文本，确认目标文本唯一或列出多个候选。
2. 用 `ControlOrMeta+F` 搜索精确片段。
3. 用截图确认命中位置和周边上下文。
4. 只选择目标段落、单元格或代码片段。
5. 粘贴替换内容。
6. 再次读取 iframe 文本和截图。

如果目标文本出现多次：

- 不要直接全局替换。
- 用标题、表格列名、前后段落或截图定位具体命中。
- 仍无法区分时询问用户选择哪个位置。

如果选区不可控：

- 对短文本，优先在原位置附近插入修订内容，并保留旧内容，等待用户确认后再删除。
- 对整块代码，先在旧代码块后创建新代码块并验证，再删除旧块。
- 对整表，先保存现表结构到 TSV 草稿，再按用户确认范围替换。

## 斜杠命令

斜杠命令适合创建表格、代码块、流程图、文本绘图、公式、目录、日期等原生块。

操作规则：

- 输入完整中文命令，例如 `/代码块`，不要用 `/dm`、`/tp` 这类短别名。
- 候选菜单出现后，读取或截图确认候选项。
- 只点击精确匹配的中文名称。
- 候选未出现、出现多个相近项或焦点离开正文时，停止并改用已展开的 `插入` 菜单或让用户手动打开入口。

禁止路径：

- 不点击 `AI 创作`。
- 不点击 `AI 听记`。
- 不使用任何 AI 生成、听记或自动改写入口。

## 菜单操作

菜单只在展开后可见文本能确认时使用。

流程：

1. 点击 `插入`、`正文`、`默认`、`菜单`、`添加图标` 或 `添加封面`。
2. 读取 iframe 文本或截图确认展开项。
3. 点击精确可见文本。
4. 操作后再次确认菜单关闭或组件出现。

失败判断：

- 点击后没有新菜单文本。
- 截图显示菜单未打开。
- 点击改变了正文光标位置。
- 出现与目标无关的浮层。

失败后不要连续盲点同一区域；改用斜杠命令、键盘路径或请用户手动打开菜单。

## 保存与验收

每次在线修改后执行验收，不能只看一次粘贴成功。

基础验收：

- iframe 文本包含新增或修改后的关键内容。
- 目标内容出现在预期上下文附近。
- 原有周边内容仍存在。
- 页面显示 `已保存`、`上次编辑`、保存中断提示消失，或刷新后内容仍在。

布局验收：

- 标题层级、表格边界、代码块、图形块、公式、目录、分栏、高亮块、封面和图标必须截图确认。
- 表格要确认列没有合并进同一个单元格。
- 代码要确认缩进和换行保留。
- 目录要确认来源标题已经是 `标题1` 到 `标题6`。

如果保存状态不可见：

- 等待数秒后重新读取文本。
- 截图保存状态区域。
- 必要时刷新页面确认内容仍在。
- 不能证明已保存时向用户说明不确定点。

## 风险分级

低风险，可按用户明确请求直接执行：

- 读取、总结、追加短段落、修改单个错别字、插入普通段落。

中风险，执行前说明假设：

- 追加长文、插入表格或代码块、设置标题层级、插入目录、替换单个段落。

高风险，执行前先确认计划和范围：

- 整篇替换。
- 删除大块内容。
- 替换整张表或多个相似段落。
- 修改图标、封面、布局、视图或整篇格式。
- 上传本地文件或图片。
- 选择企业文档、知识库、项目、应用或人员。
- 创建日程、会议、任务、投票或任何可能通知他人的组件。
- 修改分享、权限或外部可见性。

## 故障恢复

常见失败和下一步：

- iframe 不可读：检查登录、权限、企业选择、加载失败；让用户处理阻塞。
- 插入点丢失：让用户把光标放到目标位置，或改为追加到文档末尾。
- 菜单不展开：改用完整中文斜杠命令。
- 斜杠候选不稳定：打开 `插入` 菜单并基于可见项选择。
- 粘贴位置错误：立即撤销一次，重新定位后再粘贴；撤销前截图或读取文本确认错误范围。
- 表格粘贴成单元格内纯文本：撤销，改为先插入足够行列的表格，再从左上角单元格粘贴 TSV。
- 代码块变成普通段落：撤销，先创建 `代码块`，确认块出现后再粘贴。
- 自动保存不明确：等待、读取、截图，必要时刷新确认；仍不明确则报告风险。

不要在同一个失败路径上重复尝试超过两次。第二次失败后切换策略或请求用户协助建立明确的页面状态。
