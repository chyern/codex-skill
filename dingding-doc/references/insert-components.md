# 插入组件操作手册

本文件覆盖 `插入` 菜单中已验证可见的非 AI 组件。只根据当前页面实际展开的菜单或面板继续操作；不要使用 `AI 创作` 或 `AI 听记`。

## 通用规则

- 优先使用中文完整斜杠命令；菜单路径仅在菜单已经稳定展开时使用。
- 需要上传、选择企业资源、提及人员、创建会议/任务/投票、嵌入网页或连接应用时，必须确认用户授权的具体对象。
- 插入后读取 iframe 文本并截图确认组件落在目标位置。
- 如果组件打开了选择器、上传器或应用授权页，不要猜测下一步；只按可见文本继续。

## 图片、文件、在线文档

入口：

- `插入` -> `图片`
- `插入` -> `文件`
- `插入` -> `在线文档`

操作方式：

1. 确认插入位置。
2. 确认用户给出的本地文件、图片或在线文档目标。
3. 打开对应入口。
4. 在上传/选择器面板中只选择用户授权的文件或文档。
5. 插入后截图确认预览、文件名或文档卡片。

注意：

- 上传本地文件前必须有具体文件授权。
- 选择企业文档前确认文档名称或范围，避免误选。

## 同步块和模板内容

入口：

- `插入` -> `同步块`
- `插入` -> `模板内容`

使用场景：

- 同步块：同一段内容需要在多个位置复用并保持同步时使用。
- 模板内容：插入当前租户或文档提供的模板片段。

操作方式：

1. 确认目标位置。
2. 打开对应入口。
3. 同步块：确认要新建同步内容还是选择已有同步块；只选择用户指定或当前可确认的块。
4. 模板内容：读取模板面板可见名称，只选择用户指定或确认的模板。
5. 插入后验证内容块边界和可见标题。

注意：

- 同步块可能影响多个引用位置；修改已有同步块前说明影响范围。
- 模板可能带入大量结构化内容；插入前确认模板名称和位置。

## 公式、标签、表情

入口：

- `插入` -> `公式`
- `插入` -> `标签`
- `插入` -> `表情`

操作方式：

1. 公式：准备 LaTeX 或用户给出的数学表达式，打开公式入口后粘贴或输入。
2. 标签：确认标签文本和颜色/样式可见选项，只选当前面板中存在的项。
3. 表情：只在用户要求插入表情或轻量标记时使用。

验收：

- 公式保持为公式块或 inline 公式，不只是普通文本。
- 标签/表情出现在目标段落内或目标位置附近。

## 样式布局块

入口：

- `插入` -> `高亮块`
- `插入` -> `分栏`
- `插入` -> `分割线`
- `插入` -> `素材库`

使用场景：

- 高亮块：结论、风险、提示、待办说明。
- 分栏：左右对比、方案对照、角色分工。
- 分割线：章节分隔。
- 素材库：只在用户指定素材或当前面板可确认素材时使用。

操作方式：

1. 插入组件。
2. 点击组件内部目标区域。
3. 粘贴生成内容。
4. 截图确认布局没有压缩、错位或遮挡。

## 数据和图形组件

入口：

- `插入` -> `电子表格`
- `插入` -> `白板`
- `插入` -> `脑图`
- `插入` -> `路线图`
- `插入` -> `流程图`
- `插入` -> `文本绘图`

使用场景：

- 电子表格：需要公式、筛选、较大数据表或类 Sheet 能力时使用；普通小表优先原生表格。
- 白板：自由画布、草图或流程协作。
- 脑图：树状结构、知识梳理、拆解主题。
- 路线图：阶段计划、里程碑、版本规划。
- 流程图/文本绘图：流程、架构、时序或图形化表达。

操作方式：

1. 先准备结构化源数据，例如 TSV、层级列表、Mermaid 源码或节点关系。
2. 插入对应组件。
3. 在组件编辑器中填入或创建内容。
4. 返回文档后截图确认组件块。

如果无法自动化进入内部编辑器，准备好用户可粘贴的源数据，让用户手动打开编辑器后继续。

## 协作组件

入口：

- `插入` -> `提及人`
- `插入` -> `群聊`
- `插入` -> `Agoal`
- `插入` -> `钉钉日程`
- `插入` -> `视频会议`
- `插入` -> `知识库`
- `插入` -> `信息收集`
- `插入` -> `投票`

使用规则：

- 提及人/群聊会关联人员或会话；必须确认目标对象。
- 日程/视频会议可能创建或关联日程；必须确认时间、标题、参会人和是否发送通知。
- 知识库/信息收集可能连接企业资源；必须确认目标空间或表单。
- 投票必须确认题目、选项、是否匿名、截止时间等可见设置。
- Agoal 只在用户明确要求目标/OKR 类组件时使用。

验收：

- 文档中出现对应人员、会话、日程、投票或知识库卡片。
- 未发送未授权通知，未选择错误对象。

## 项目管理组件

入口：

- `插入` -> `项目任务`
- `插入` -> `项目看板`

操作方式：

1. 确认项目空间、任务标题、负责人、截止时间、状态和描述。
2. 插入 `项目任务` 或 `项目看板`。
3. 在可见面板中只填写用户确认的字段。
4. 插入后验证任务或看板卡片出现在目标位置。

注意：

- 如果创建任务会通知负责人或写入项目系统，用户的明确请求可视为授权；否则先确认。
- 不确定项目空间时不要创建。

## 小工具和嵌入组件

入口：

- `插入` -> `目录`
- `插入` -> `表态`
- `插入` -> `日期`
- `插入` -> `投屏`
- `插入` -> `计时器`
- `插入` -> `倒计时`
- `插入` -> `手写签名`
- `插入` -> `模板按钮`
- `插入` -> `内嵌网页`
- `插入` -> `三方应用`
- `插入` -> `企业内应用`

使用方式：

- 目录：先确保文档有 `标题1` 到 `标题6`，再插入目录。
- 表态：需要收集轻量反馈时插入。
- 日期：计划、里程碑、会议纪要中插入日期。
- 投屏：只在用户要求演示或投屏场景时使用。
- 计时器/倒计时：会议、演练、评审限时场景。
- 手写签名：确认签名人和签署位置，不代替用户签名。
- 模板按钮：用户要求可重复生成固定块时使用。
- 内嵌网页：确认 URL 和可公开嵌入性，不输入敏感链接。
- 三方应用/企业内应用：可能触发应用授权或数据连接，必须确认应用和目标数据。

验收：

- 目录能反映标题层级。
- 日期、计时器、倒计时显示正确值。
- 嵌入网页或应用卡片显示目标名称/URL。
- 不触发未授权投屏、签名、应用授权或数据发送。
